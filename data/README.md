# Data

## `raw/diabetes.csv`

A **synthetic** dataset (2,000 rows) generated by [`ml/generate_data.py`](../ml/generate_data.py).
It is not real patient data. Feature distributions and the feature‚Üíoutcome
relationship are modeled after well-documented clinical patterns for the 8
standard diabetes risk factors, so the resulting model behaves realistically
for demo and learning purposes.

To regenerate it (e.g. with a different seed or sample size):

```bash
python ml/generate_data.py
```

### Columns

| Column                     | Type  | Description                                      | Typical range |
|-----------------------------|-------|--------------------------------------------------|----------------|
| `Pregnancies`               | int   | Number of times pregnant                         | 0 ‚Äì 17         |
| `Glucose`                   | float | Plasma glucose concentration (mg/dL)              | 50 ‚Äì 250       |
| `BloodPressure`              | float | Diastolic blood pressure (mm Hg)                  | 30 ‚Äì 130       |
| `SkinThickness`              | float | Triceps skinfold thickness (mm)                   | 0 ‚Äì 60         |
| `Insulin`                    | float | 2-hour serum insulin (¬µU/mL)                      | 0 ‚Äì 600        |
| `BMI`                        | float | Body mass index (kg/m¬≤)                           | 14 ‚Äì 60        |
| `DiabetesPedigreeFunction`   | float | Score estimating diabetes likelihood from family history | 0.05 ‚Äì 2.5 |
| `Age`                        | int   | Age in years                                      | 18 ‚Äì 85        |
| `Outcome`                    | int   | 1 = diabetes-positive, 0 = diabetes-negative (target) | ‚Äî          |

### Using real data instead

Replace this file with any dataset that has the same 9 columns (e.g. the
public **Pima Indians Diabetes Dataset**), then re-run:

```bash
python ml/train.py
```
